> **この記事で一言でわかること**: 「LLMの幻覚・法令違反表現を本番環境で防ぐ自動チェックの実装パターン」

# 背景

CBD系のメディアサイトをGemini APIで記事自動生成している。ある日、生成された記事に以下の一文が混ざっていた。

> 「某ハーバード大学の2023年研究によると、CBDには睡眠障害への効果があることが証明されています」

**根拠はない。** そんな研究は確認できていない。「証明されています」という断定表現も薬機法上問題がある。

この記事では、LLMの幻覚（ハルシネーション）と法令違反表現を本番環境で防ぐための実装パターンを紹介する。

# 問題1: LLMによる架空出典の生成

## なぜ起きるか

LLMは「それっぽい文章」を生成することに最適化されている。科学的な文章には出典がつく → だから出典らしきものを生成する。

確認された架空出典パターン：

```
1. 「某○○大学」系: 某ハーバード大学、某東京大学医学部...
2. 「著名ジャーナル」系: NEJM (2024)、Lancet研究...（実在するが論文は存在しない）
3. 「自作調査」系: 「当サイト独自調査によると87%が...」
4. 「年付き統計」系: 「2024年のデータでは〜%が...」（数字を創作）
```

## 検出実装

```python
import re

SUSPICIOUS_CITATION_PATTERNS = [
    # 「某○○大学/研究所」パターン
    (r'某[^\s。、]{1,15}(?:大学|研究所|医療機関)', '研究によると'),
    # 「当サイト調査」パターン
    (r'当サイト(?:独自)?調査', '各種調査'),
    # 特定数値の自作統計パターン（高精度な%は要確認）
    (r'([89]\d\.\d%).*?(?:が|の方)', r'\1程度（要出典確認）'),
]

def check_suspicious_citations(html: str) -> tuple[str, list[str]]:
    """
    架空出典を検出・置換する。
    Returns: (修正済みHTML, 警告メッセージリスト)
    """
    warnings = []
    
    for pattern, replacement in SUSPICIOUS_CITATION_PATTERNS:
        matches = re.findall(pattern, html)
        if matches:
            warnings.append(f"架空出典の可能性: {matches}")
            html = re.sub(pattern, replacement, html)
    
    return html, warnings
```

警告が出た記事には Google Sheets の承認画面でフラグを立てる：

```python
if warnings:
    sheets.update_cell(row, STATUS_COL, '要確認')
    sheets.update_cell(row, WARNING_COL, '\n'.join(warnings))
```

# 問題2: 薬機法違反表現

## 対象ジャンル

CBD・サプリメント・スキンケア・健康食品を扱うサイトでは、以下の表現が薬機法（医薬品、医療機器等の品質、有効性及び安全性の確保等に関する法律）に抵触する可能性がある。

| 違反表現 | 理由 |
|:---|:---|
| 「○○に効きます」 | 医薬品的な効能効果の標榜 |
| 「○○が治ります」 | 疾病の治癒を示唆 |
| 「奇跡の○○」 | 誇大広告 |
| 「医師も認める」 | 医師の推薦を利用した誇大表現 |

## 自動修正実装

```python
YAKKI_FORBIDDEN_PATTERNS = [
    # 治癒・治療系
    (r'(?:治る|治す|治療(?:する|できる|に効果))', '改善が期待される'),
    # 効能断定系
    (r'(?:効く|効果がある|効果を(?:発揮|示))', '効果が期待されている'),
    # 誇大表現系
    (r'(?:万能|奇跡の|医師も認める)', '注目されている'),
    # 病名 + 効果の組み合わせ
    (r'(うつ|不眠|不安|疼痛)(?:に効|を治|が改善)', r'\1への働きかけが研究されて'),
]

def check_yakki_compliance(html: str) -> tuple[str, list[str]]:
    """
    薬機法違反表現を検出・自動修正する。
    Returns: (修正済みHTML, 修正ログリスト)
    """
    modifications = []
    
    for pattern, replacement in YAKKI_FORBIDDEN_PATTERNS:
        original_matches = re.findall(pattern, html)
        if original_matches:
            modifications.append(f"薬機法修正: {original_matches} → {replacement}")
            html = re.sub(pattern, replacement, html)
    
    return html, modifications
```

## プロンプト側でも事前に防ぐ

自動修正は後処理だが、プロンプトで事前に防ぐほうが品質が高い。

```python
YAKKI_INSTRUCTION = """
【薬機法・景表法遵守の絶対ルール】
以下の表現は使用禁止:
- 「治る」「治す」「治療」「効く」「効果がある」「万能」「奇跡」
- 断定的な医療効果の表現

代わりに以下の表現を使うこと:
- 「〜の可能性が研究で示されている」
- 「〜という報告がある」
- 「〜への効果が期待されている」（あくまで期待・可能性の表現）

出典は実在するもののみ引用。架空の大学名・研究は絶対に書かないこと。
"""
```

# パイプライン全体でのチェックフロー

```
Gemini API 記事生成
  ↓
check_yakki_compliance()   # 薬機法チェック・自動修正
  ↓
check_suspicious_citations()  # 架空出典チェック・置換
  ↓
Google Sheets に書き込み
  ↓
[人間の承認]  ← ← ← 警告フラグがある行はオレンジ表示
  ↓
WordPress 投稿
```

# 実際に防げた事故の例

| カテゴリ | 生成内容 | 処理結果 |
|:---|:---|:---|
| 薬機法 | 「CBDは不眠症に効きます」 | 「不眠への働きかけが研究されています」に自動修正 |
| 架空出典 | 「某ハーバード大学の研究では」 | 「研究によると」に置換 + 要確認フラグ |
| 誇大表現 | 「奇跡のオイル」 | 「注目されているオイル」に自動修正 |

# 限界と残課題

自動チェックで防げないもの：

1. **文脈依存の判断**: 「効果的な使い方」は問題ないが「○○に効果的」は要確認
2. **新しいパターンの嘘**: LLMは創造的に嘘をつく。パターン定義は継続的な更新が必要
3. **外国語・表記ゆれ**: カタカナ・漢字・ひらがな混在のパターンは完全な網羅が難しい

結局のところ、自動チェックは「明らかな違反の自動修正」に留め、人間の最終確認を省略しないことが重要だ。

# まとめ

| 対策 | 実装 | 効果 |
|:---|:---|:---|
| プロンプト指示 | 禁止表現リストをプロンプトに含める | 生成段階での予防 |
| 後処理チェック | 正規表現で自動修正・警告 | 漏れの自動修正 |
| 承認フロー | 警告フラグ付きで人間がレビュー | 最終防衛ライン |
